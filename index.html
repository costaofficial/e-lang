What is E

E is a language that describes when to do something, on what, and what to do. The runtime handles retries, waits, errors — you just describe the intent.

Declarative. Event-oriented. Human-readable.

◆ ◇ ◆ ◇ ◆
Example
// every day at 2am — export Garmin data
time every day at 02:00 do
    with browser do
        open "https://connect.garmin.com"
        with page { timeout: 10s } do
            login "user" "pass" or stop
            click "#export-all"
            wait download
        done
    done
done or log error
Features
⏰ Time
Schedule scripts with time every day at — APScheduler underneath.
🌐 Browser
Automate the web: open, find, click, wait via Playwright.
🔄 Retry
Built-in retry N times do ... done with exponential backoff.
⚠️ Fallback
Every statement can have an or fallback — local or global.
📁 Files
write, create, run — real filesystem and subprocess.
📧 Email
email to "addr" file "report.pdf" — SMTP ready.
Try it
$ git clone https://github.com/costaofficial/e-lang.git
$ cd e-lang
$ python3 runtime/run_e.py examples/hello.e
