What is E

E is a general-purpose language. You describe when to do something, on what, and what to do. The runtime handles the complexity.

Declarative. Human-readable. Event-oriented. Single binary, zero dependencies.

◆ ◇ ◆ ◇ ◆
Example
// 3-tier .eee file: plugins + logic + UI
:sys
use "http"
use "db"

:core
fn save_user name do
    let data = sys_call "http" "e_get" "https://api.example.com"
    sys_call "db" "e_query" "users" "INSERT INTO data VALUES (data)"
    log "saved"
done

:ui
<h1>App</h1>
<script>console.log('ready');
Features

🌐 Browser

open, find, click, login, download via headless Chrome.

⏰ Scheduling

time every day at 02:00 do with --watch.

🖥️ WebView

:ui renders HTML/JS in native window.

🔌 Plugins

json, fs, db, http built-in. More via sys_call.

📦 Single binary

Zero dependencies. One e file, everything included.

📁 Scripting

Shell commands, file I/O, variables, functions, retry.

Quick start
curl -L https://github.com/costaofficial/e-lang/releases/latest/download/e -o e
chmod +x ./e
./e examples/hello.eee

# with browser, plugins, scheduling
./e --live examples/plugins.eee
./e --watch --live examples/g.eee
